Guided project: Habit Garden App

Build a simple but complex-enough app: users add habits, track streaks, and see progress charts. This teaches API design, frontend state, validation, and deployment.

  1. 1. Scope: write a one-page product spec in plain language.
  2. 2. Prompt: ask model for architecture + folder structure.
  3. 3. Scaffold: generate React + Tailwind UI with mobile-first layout.
  4. 4. Backend: add simple REST API for habits and streak logs.
  5. 5. Test: auto-generate tests, then manually verify each user flow.
  6. 6. Ship: deploy and ask model for launch checklist + bug triage.

Prompt templates

You are my senior software coach.
My level: absolute beginner.
Goal: build a Habit Garden app with React + API.
Constraints: keep files small, explain every command, test each step.
Return: (1) plan, (2) exact commands, (3) code snippets, (4) checkpoints.

Simple weekly learning plan

  • Day 1: Understand prompt anatomy and project scoping.
  • Day 2: Generate UI and improve spacing/typography.
  • Day 3: Add backend routes and data validation.
  • Day 4: Connect frontend to backend, handle errors.
  • Day 5: Write tests and performance checks.
  • Day 6: Deploy and run a bug bash.
  • Day 7: Reflect, refactor, and document.
